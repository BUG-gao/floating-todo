# 悬浮待办产品使用文档

## 产品定位

悬浮待办是一款 macOS 桌面小组件，用一个轻量浮窗管理今天、明天、后天的待办事项，并支持全局备忘录、透明度、桌面可见性、置顶和背景色设置。

## 核心使用场景

- 处理当天短任务：例如整理收件箱、归档笔记、整理会议纪要。
- 提前规划未来两天：例如把“明天提交方案”和“后天复盘项目”放到对应日期。
- 记录临时信息：例如待办备注、参考链接、临时想法。
- 在其他 App 上方保持轻量提醒：例如写文档时保持待办浮窗可见。

## 浮悬待办小组件

小组件默认显示在桌面上方，包含标题、未完成数量、日期切换、待办列表、添加输入框和设置按钮。

示例：上午打开电脑后，在“今天”里添加三条待办：

- 整理收件箱
- 整理需求清单
- 归档笔记

完成后点击圆形勾选按钮，已完成事项会被移动到未完成事项下方。

## 日期分组

小组件支持三个固定日期分组：

- 今天
- 明天
- 后天

每个分组都有独立待办列表。切换日期不会影响其他分组内容。

示例：

- 今天：整理收件箱
- 明天：设计悬浮窗视觉
- 后天：整理需求清单

## 添加与编辑待办

在底部输入框输入内容后提交，即可添加到当前日期分组。空白内容会被忽略。双击非紧凑模式下的待办标题，可回到底部输入框进行编辑。

示例：

输入“10:00 日程确认”，提交后会出现在当前日期的列表顶部。

## 待办详情

每条待办可以添加详情描述。详情适合放参考链接、备注、上下文说明。

示例：

标题：10:00 日程确认

详情：https://example.com/task-note

## 移动和删除

待办可通过更多菜单移动到其他日期，也可以删除。

示例：

如果“整理需求清单”今天来不及做，可以通过菜单移到“后天”。

## 全局备忘录

备忘录是一个全局便签，不跟随今天、明天、后天切换。它适合记录临时想法、链接和碎片信息。

备忘录默认关闭。用户在设置中开启后，底部抽屉默认展开。用户可以手动收起或展开，收起状态会被记住。编辑区使用多行输入，内容会自动保存。

示例：

在全局备忘录中记录：

- 参考编号 0000
- 临时参考链接
- 今天沟通时提到的产品想法

无论当前选中“今天”“明天”还是“后天”，看到的都是同一份备忘录。

## 背景色设置

用户可以在小组件设置里自定义整体背景色。背景色会影响整个浮窗的主背景基调，内部控件、待办行、输入框和备忘录抽屉仍保留统一透明材质。

设置里提供恢复默认按钮，点击后回到默认蓝白或深色背景。

示例：

如果用户希望待办浮窗更贴近桌面壁纸，可以选择浅绿色、浅紫色或暖灰色作为整体背景。

## 外观与透明度

外观支持跟随系统、浅色、深色。透明度滑块用于调整浮窗整体透明度，适合在不同桌面环境下减少遮挡。

示例：

写文档时把透明度降低一些，让浮窗不遮住正文；任务密集时提高透明度，便于快速阅读。

## 始终置顶与所有桌面可见

始终置顶开启后，小组件保持在普通窗口上方。显示在所有桌面开启后，切换 macOS Space 时小组件仍可见。

示例：

用户在浏览器、文档编辑器和聊天软件之间切换时，待办小组件仍保持可见。

## Option 操作小组件

设置里提供 Option 操作小组件相关选项。开启后，按住 Option 点击浮窗覆盖区域时仍操作小组件；关闭时，Option 点击会落到下层窗口。低透明度自动允许 Option 操作可在透明度较低时自动启用这一行为。

示例：

当浮窗覆盖了下层 App 的按钮，用户可以用 Option 点击决定是操作小组件还是让点击落到下层窗口。

## 推荐工作流

1. 早上把今天最重要的 3 到 5 件事写进“今天”。
2. 把明确的未来事项放入“明天”或“后天”。
3. 用全局备忘录保存临时链接和会议号。
4. 根据当前工作环境调整透明度和背景色。
5. 下班前清理已完成任务，把未完成事项移动到合适日期。

## 常见问题

问：备忘录会跟日期切换吗？

答：不会。备忘录是全局便签，始终只有一份。

问：关闭备忘录会删除内容吗？

答：不会。关闭只是隐藏入口，内容仍会保留。

问：恢复默认背景色会影响待办数据吗？

答：不会。它只恢复视觉背景，不影响待办和备忘录内容。

问：为什么只有今天、明天、后天？

答：产品定位是轻量短期执行，不是完整项目管理工具。三天范围可以减少规划负担。
