## 📖 这是什么

很多待办 App 太「重」：要登录、要分项目、要切换窗口才能看到。
**悬浮待办** 反其道而行——它是一个**永远飘在桌面上、半透明、不打扰**的小浮窗：

- 你在写文档、开会、敲代码时，它就静静停在角落，**一眼能看到今天要做什么**；
- 不需要切窗口、不需要登录、不需要联网，**所有数据只存在你自己的电脑里**；
- 只管「今天 / 明天 / 后天」三天，**专注短期执行，不做复杂的项目管理**，减少规划负担。

适合：处理当天短任务、提前规划未来两天、随手记会议号 / 临时链接、在其他 App 上方保持轻量提醒。

## ✨ 功能一览

### 待办管理
- 📅 **今天 / 明天 / 后天** 三个独立列表，互不干扰。
- ➕ 快速新增、✅ 一键完成、🗑️ 删除，完成的事自动沉到底部并保留你的手动排序。
- ✏️ 双击标题即可编辑；三点菜单可**编辑、加描述、移到其他日期、删除**。
- 📝 每条待办可加**描述**（会议链接、备注等），**详情里的网址自动识别成可点击链接**，直接点开。
- 🔁 **每天常驻待办**：把「梳理货源」「复盘」这类每天都要做的事「设为每天」，它会固定显示在今天列表顶部，**跨天不消失、每天自动重置为未完成**。
- 🌅 **跨天自动滚动**：过了零点，明天自动变今天、后天变明天；昨天没做完的事顺延到今天底部，不丢工作。
- 🧹 **一键清除已完成**，列表不堆积。

### 窗口与外观
- 📌 **始终置顶**，可显示在**所有桌面空间**（切换 Space 也能看到）。
- 🫧 **半透明**，透明度可调，写文档时调低不挡正文，任务多时调高方便看。
- 🎨 支持**自定义背景色**，以及**跟随系统 / 浅色 / 深色**三种外观。
- 🖱️ 边缘 / 四角**拖拽缩放**；缩到最小后右上角有**一键放大**按钮。
- 🔽 **紧凑模式**：窗口缩小到一定程度只显示今天的内容，最省地方。
- 🧭 **状态栏小组件**：不占程序坞和 ⌘Tab，入口在右上角菜单栏。
- 👆 **Option 点击穿透**：浮窗挡住下层按钮时，可按住 `Option` 让点击落到下层窗口（可在设置里开关）。

### 备忘录
- 🗒️ **全局备忘录**抽屉：一份不随日期切换的便签，适合记会议号、临时链接、突发想法；内容自动保存。

## ⚠️ 重要：首次打开提示「已损坏」/「无法验证开发者」怎么办

本应用**没有购买 Apple 付费开发者证书做签名和公证**（一个独立免费项目，签名需 99 美元/年）。
因此 macOS 的 Gatekeeper 在首次打开时会拦截，可能提示：

> *「"悬浮待办" 已损坏，无法打开。」* 或 *「无法打开，因为它来自身份不明的开发者。」*

**这不是病毒，是因为没签名。** 任选以下一种方法解决：

### 方法 A：右键打开（最简单）
1. 在「应用程序」里找到 **悬浮待办**；
2. **右键（或按住 Control 单击）→ 打开**；
3. 在弹窗里再点一次「打开」。之后就能正常使用了。

### 方法 B：去掉「隔离」属性（提示「已损坏」时用这个）
打开「终端」执行（按提示输入开机密码，密码不会显示是正常的）：

```bash
sudo xattr -dr com.apple.quarantine "/Applications/悬浮待办.app"
```

然后正常双击打开即可。

### 方法 C：系统设置里放行
打开后被拦的话，到 **系统设置 → 隐私与安全性**，
下滑找到「已阻止"悬浮待办"」，点 **仍要打开**。

## 🛠️ 技术栈

- **语言**：Swift 6.2
- **UI**：SwiftUI + AppKit（`NSPanel` 无边框浮窗）
- **持久化**：本地 `UserDefaults`（JSON 快照，纯本地、不联网）
- **架构**：`ToDoCore`（模型 / 状态 / UI）+ `ToDoModule`（可执行入口）+ XCTest 单元测试
- **测试**：`swift test`，覆盖增删改、完成排序、跨日期移动、跨天滚动、常驻待办、持久化与旧数据兼容等 27 个用例

构建脚本：

- `scripts/install-local.sh`：构建并安装到 `/Applications`
- `scripts/build-installer.sh`：生成 `.pkg` 安装包
- `scripts/build-dmg.sh`：生成 `.dmg` 镜像
